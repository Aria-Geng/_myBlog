HTML与CSS

DOCTYPE(文档类型) 的作用：

  • DOCTYPE是HTML5中的文档类型声明，它必须声明在HTML⽂档的第⼀⾏，它的目的是告诉浏览器（解析器）应该以什么样（html或xhtml）的文档类型来定义和解析文档。

  • 拓展 ： !doctype html 的作用就是让浏览器进入标准页面渲染模式，使用W3C的标准解析渲染页面，浏览器以其支持的最高标准呈现页面；如果不写，浏览器就会进入怪异页面渲染模式，浏览器使用自己的怪异模式解析渲染页面，页面以一种比较宽松的向后兼容的方式显示，我们需要避免此类情况发生。

行内元素、块级元素、空元素

  • 行内元素：a b span img input select strong；
  • 块级元素：div ul ol li dl dt dd h1 h2 h3 h4 h5 h6 p；
  • 空元素有：<br>、<hr>、<img>、<input>、<link>、<meta>；

iframe标签的作用？

iframe是用来在网页中嵌入另一个网页的标签，ifram标签上有一个src属性接收的就是另一个网页的页面地址，

常见的布局方式

  1. flex布局（弹性布局）：主要用来作移动页面适配不同的屏幕大小，让浏览自动划分区域
  2. grid布局：类似flex布局，但是grid是利用行和列将一个盒子划分成一个二维的区域，也有点类似与栅格系统，利用grid可以实现flex能够实现的一切
  3. 流式布局（百分比布局）：利用百分比和浮动划分区域大小和布局展示，超出的部分自动换行显示
  4. 响应式布局：利用html5的媒体查询功能，更具屏幕不同的宽度来指定不同的样式，实现一个网站多个设备显示
  5. 多栏布局（栅格系统）：利用百分比+浮动的方式划分页面，通常一些ui组件库都会提供栅格系统

src和href的区别

  • 相同点：都是用来引入外部资源；
  • 不同点
    • 请求资源类型不同
      • href表示超文本引用，建立当前元素和文档之间的链接，比如link、a。
      • 请求 src 资源时会将其指向的资源下载并应用到文档中，比如script，img 、iframe；
    • 作用结果不同
      • href 用于在当前文档和引用资源之间确立联系；
      • src 用于替换当前内容；
    • 浏览器解析方式不同
      • 在文档中添加href ，浏览器识别文档然后并行下载资源，而且不会停止对当前文档的处理。
      • 浏览器解析到src ，会暂停其他资源的下载和处理，直到将该资源加载、编译、执行完毕。

flex=1

  • flex ：1 即为 flex-grow ：1。经常用作自适应布局，将父容器的 display：flex，侧边栏大小固定后，将内容区 flex：1，内容区则会自动放大占满剩余空间。

  • flex:1和flex:auto的区别

    • flex:1

    • flex:auto

  • flex属性是flex-grow、flex-shrink和flex-basis的简写，默认值为0 1 auto，（后两个属性可选）。

  • 当一个容器设置display:flex变成一个flex容器后，如果容器没有被占满，换言之有剩余空间，则flex-grow起作用。若空间不足，则flex-shrink起作用。在计算放大或缩小比例时，根据flex-basis的值来计算比例。

    • flex-grow：定义项目的的放大比例
      • 默认为 0，即 即使存在剩余空间，也不会放大；
      • 所有项目的 flex-grow 为 1：等分剩余空间（自动放大占位）；
      • flex-grow 为 n 的项目，占据的空间（放大的比例）是 flex-grow 为 1 的 n 倍。
    • flex-shrink：定义项目的缩小比例
      • 默认为 1，即 如果空间不足，该项目将缩小；
      • 所有项目的 flex-shrink 为 1：当空间不足时，缩小的比例相同； flex-shrink 为 0：空间不足时，该项目不会缩小； flex-shrink 为 n 的项目，空间不足时缩小的比例是 flex-shrink 为 1 的 n 倍。
    • flex-basis：定义在分配多余空间之前，项目占据的主轴空间（main size），浏览器根据此属性计算主轴是否有多余空间
      • 默认值为 auto，即 项目原本大小；
      • 设置后项目将占据固定空间。

圣杯布局和双飞翼布局的异同点

  • 相同之处
    • 需求相同
      • 两侧内容宽度固定，中间内容宽度自适应
      • 中间结构在DOM上优先，以便于优先渲染
    • 都使用了float浮动向左脱离文档流，让左中右三列浮动，通过父外边距形成三列布局。
  • 不同之处
    • 实现方法的不同：
      • 圣杯布局通过float搭建布局+margin使三列布局到一行上+relative相对定位调整位置。给外部容器加padding，通过相对定位把两边定位出来。
      • 双飞翼布局是通过float+margin，没有使用相对定位。靠在中间这层外面套一层div加padding将内容挤出来中间。

如何让一个元素垂直水平都居中 ？

  • flex：给父元素添加display: block; 给子元素添加margin: auto;
  • flex：给父元素添加display: flex; justify-content: center; align-items: center
  • 定位：子元素是固定宽高（position: absolute, top:0; left:0; right:0; bottom: 0;）
  • 定位：子元素position: absolute; top: 50%; left:50%; transform: translate(-50%, -50%);
  • 将父元素转换成display: table-cell;

css中哪些样式可以继承？

大部分可以继承的样式都是文字样式，行高，颜色，大小，字体，….

position的属性和作用

  • absolute：绝对定位，相对于父级有定位的元素定位，元素自身的位置不会保留（脱离文档流）
  • relative：相对定位，相对于元素自身定位，元素自身的位置会保留
  • fixed：固定定位, 相当于浏览器的屏幕进行定位（脱离文档流）
  • sticky：粘性定位，粘性定位是相对定位和固定定位的混合，元素在浏览滚动条内是不会发生移动的，当浏览器滚动超过当前元素位置，当前元素改为固定定位跟谁浏览器移动，一般用来是实现图钉效果

BFC

  • 块级格式化上下文。

  • 满足下列条件之一就可触发BFC

    • 根元素，即html

    • float的值不为none（默认）

    • overflow的值不为visible（默认）

    • display的值为table-cell, table-caption, inline-block, flex, 或者 inline-flex 中的其中一个

    • position的值为absolute或fixed

css样式的优先级

  1. !important：覆盖页面内任何位置定义的元素样式。ie6不支持该属性。
  2. 内联样式：html中给元素标签加style。造成css难以管理，不推荐使用。
  3. 由一个或多个id选择器定义。例如，#id{margin:0;}会覆盖.classname{margin:3pxl}
  4. 由一个或多个类选择器、属性选择器、伪类选择器定义。如.classname{margin:3px}会覆盖div{margin:6px;}
  5. 由一个或多个类型选择器定义。如div{marigin:6px;}覆盖{margin:10px；}
  6. 通配选择器，如{marigin:6px;}
  • 行内样式（style=”…”）>ID 选择器(#box{…})>类选择器(.con{…})>标签选择器(dic{…})>通用选择器(*{…})
  • 行内样式（1000）>ID选择器（100）>类选择器（10）>标签选择器（1）>通用选择器（0）
    • 通用选择器（*），子选择器（>），和相邻同胞选择器（+）的权值为0

css选择器

  • id选择器、class选择器、元素选择器、伪类选择器

  • :hover、:link、:active、:visited、:after、:before

  • :nth-child、:frist-child、:last-child、:disabled、:checked

  • 子代选择器 (ul > li)、后代选择器 (ul li)、属性选择器(a[class], a=[class=”hover”])

  • 相邻选择器(ul + div) ：后面一个、兄弟选择器(ul ~ div)：后面所有

css中常见的单位

  • px：正常的像素大小，但是在高清显示屏上1px可能代表4个
  • rem：相对html的font-size大小，比如html的font-size：100px；那么1rem就是100px
  • em：和rem一样，只不过em是相对于自身的font-size
  • vw、vh：类似于百分比，vw和vh始终都相对于屏幕的大小，100vw就相当屏幕的100%
  • %：相对于父元素有大小才会启作用的单位

css中隐藏一个元素

  • display：none; 隐藏一个元素，元素会消失在渲染树中，在页面中也不会占位
  • visibity:hidden; 隐藏一个元素，但是该元素所占据的位置还在
  • opcity:0; 将元素变成透明，元素位置还在，元素如果有事件依然可点击

css清除浮动方法

  • clear: both;
  • 在父元素使用overflow: hidden;
  • 使用伪元素::before、::after

移动端1像素边框

  • 通过伪元素插入一个高度为1px，宽度200%的元素，通过transform：scale（50%）进行缩放，可以实现移动端1px边框

transition和animation的区别

  • transition：过渡动画，从一个值到另一个值直接动画过渡

  • animation：关键帧动画，通过添加帧实现多个值之间更精准的动画效果

script标签上添加async或者defer属性的作用

  • 正常情况下script标签加载脚本会阻塞浏览器渲染，等到js都加载完成后继续渲染，所以一般情况下把script标签放到页面的底部
  • 当添加defer或者async属性后就不会阻塞浏览器渲染，变成了异步加载
    • async：表示当前引入的js是异步执行，如果加载好，就会开始执行，执行会阻塞页面的渲染
    • defer：表示延迟执行引入js，不会阻塞页面的渲染。js的加载是异步的，延迟到页面解析完成后才执行

页面统计数据中，常用的 PV、UV 指标分别是什么？

  • PV(页面访问量)：同一用户多次浏览也会累加；
  • UV(独立访客量)：通过互联网访问、浏览这个网页的人。访问网站的一台电脑客户端为一个访客。 00:00-24:00内相同的客户端只被计算一次。

浏览器的标准模式和怪异模式的由来和区别

  • 出现背景：HTML和CSS标准之前，浏览器对html和css的解析有各自不同的方式，很多旧版本的网页都是按照非标准的模式实现的页面，在HTMl和CSS标准定义完成后，浏览器开始按照w3c规定标准来解析页面，还需要保证之前的页面能够正常显示，所以浏览器都会有两种渲染模式：标准模式和怪异模式，在标准模式下，浏览器按照w3c提供的标准解析和渲染页面，在怪异模式下，浏览按照非标准的方式对文档进行解析渲染，
  • 浏览会根据html doctype确定使用哪种模式执行页面的渲染
  • 区别
    • 大小计算法方式
      • 盒子模型
        • 标准模式下IE盒模型（content（宽高） + 填充padding + 边框border + 边距margin）
        • 怪异模式下（content + maring）而content区域包括了（content，padding，border）
        • 在css3中可以通过 box-sizing来控制盒模型content-box, padding-box, border-box
      • 行内元素的高度：标准模式下不生效，怪异模式下会生效
      • margin： 在标准模式下margin: 0 auto; 会水平居中，怪异模式下不会
  • 拓展 ： !doctype html 的作用就是让浏览器进入标准页面渲染模式，使用W3C的标准解析渲染页面，浏览器以其支持的最高标准呈现页面；如果不写，浏览器就会进入怪异页面渲染模式，浏览器使用自己的怪异模式解析渲染页面，页面以一种比较宽松的向后兼容的方式显示，我们需要避免此类情况发生。

  • flex:1和flex:auto的区别

    • flex:1

    • flex:auto

  • flex属性是flex-grow、flex-shrink和flex-basis的简写，默认值为0 1 auto，（后两个属性可选）。

  • 当一个容器设置display:flex变成一个flex容器后，如果容器没有被占满，换言之有剩余空间，则flex-grow起作用。若空间不足，则flex-shrink起作用。在计算放大或缩小比例时，根据flex-basis的值来计算比例。

    • flex-grow：定义项目的的放大比例
      • 默认为 0，即 即使存在剩余空间，也不会放大；
      • 所有项目的 flex-grow 为 1：等分剩余空间（自动放大占位）；
      • flex-grow 为 n 的项目，占据的空间（放大的比例）是 flex-grow 为 1 的 n 倍。
    • flex-shrink：定义项目的缩小比例
      • 默认为 1，即 如果空间不足，该项目将缩小；
      • 所有项目的 flex-shrink 为 1：当空间不足时，缩小的比例相同； flex-shrink 为 0：空间不足时，该项目不会缩小； flex-shrink 为 n 的项目，空间不足时缩小的比例是 flex-shrink 为 1 的 n 倍。
    • flex-basis：定义在分配多余空间之前，项目占据的主轴空间（main size），浏览器根据此属性计算主轴是否有多余空间
      • 默认值为 auto，即 项目原本大小；
      • 设置后项目将占据固定空间。

圣杯布局和双飞翼布局的异同点

  • 相同之处
    • 需求相同
      • 两侧内容宽度固定，中间内容宽度自适应
      • 中间结构在DOM上优先，以便于优先渲染
    • 都使用了float浮动向左脱离文档流，让左中右三列浮动，通过父外边距形成三列布局。
  • 不同之处
    • 实现方法的不同：
      • 圣杯布局通过float搭建布局+margin使三列布局到一行上+relative相对定位调整位置。给外部容器加padding，通过相对定位把两边定位出来。
      • 双飞翼布局是通过float+margin，没有使用相对定位。靠在中间这层外面套一层div加padding将内容挤出来中间。

position的属性和作用

  • absolute：绝对定位，相对于父级有定位的元素定位，元素自身的位置不会保留（脱离文档流）
  • relative：相对定位，相对于元素自身定位，元素自身的位置会保留
  • fixed：固定定位, 相当于浏览器的屏幕进行定位（脱离文档流）
  • sticky：粘性定位，粘性定位是相对定位和固定定位的混合，元素在浏览滚动条内是不会发生移动的，当浏览器滚动超过当前元素位置，当前元素改为固定定位跟谁浏览器移动，一般用来是实现图钉效果

css样式的优先级

  1. !important：覆盖页面内任何位置定义的元素样式。ie6不支持该属性。
  2. 内联样式：html中给元素标签加style。造成css难以管理，不推荐使用。
  3. 由一个或多个id选择器定义。例如，#id{margin:0;}会覆盖.classname{margin:3pxl}
  4. 由一个或多个类选择器、属性选择器、伪类选择器定义。如.classname{margin:3px}会覆盖div{margin:6px;}
  5. 由一个或多个类型选择器定义。如div{marigin:6px;}覆盖{margin:10px；}

css中常见的单位

  • rem：相对html的font-size大小，比如html的font-size：100px；那么1rem就是100px

transition和animation的区别

  • transition：过渡动画，从一个值到另一个值直接动画过渡
  • animation：关键帧动画，通过添加帧实现多个值之间更精准的动画效果

script标签上添加async或者defer属性的作用

  • 正常情况下script标签加载脚本会阻塞浏览器渲染，等到js都加载完成后继续渲染，所以一般情况下把script标签放到页面的底部
  • 当添加defer或者async属性后就不会阻塞浏览器渲染，变成了异步加载
    • async：表示当前引入的js是异步执行，如果加载好，就会开始执行，执行会阻塞页面的渲染
    • defer：表示延迟执行引入js，不会阻塞页面的渲染。js的加载是异步的，延迟到页面解析完成后才执行

浏览器的标准模式和怪异模式的由来和区别

  • 出现背景：HTML和CSS标准之前，浏览器对html和css的解析有各自不同的方式，很多旧版本的网页都是按照非标准的模式实现的页面，在HTMl和CSS标准定义完成后，浏览器开始按照w3c规定标准来解析页面，还需要保证之前的页面能够正常显示，所以浏览器都会有两种渲染模式：标准模式和怪异模式，在标准模式下，浏览器按照w3c提供的标准解析和渲染页面，在怪异模式下，浏览按照非标准的方式对文档进行解析渲染，
  • 浏览会根据html doctype确定使用哪种模式执行页面的渲染
  • 区别
    • 大小计算法方式
      • 盒子模型
        • 标准模式下IE盒模型（content（宽高） + 填充padding + 边框border + 边距margin）
        • 怪异模式下（content + maring）而content区域包括了（content，padding，border）
        • 在css3中可以通过 box-sizing来控制盒模型content-box, padding-box, border-box
      • 行内元素的高度：标准模式下不生效，怪异模式下会生效
      • margin： 在标准模式下margin: 0 auto; 会水平居中，怪异模式下不会
